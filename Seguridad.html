Seguridad

Medida de capacidad del sistema para proteger datos. La seguridad es muy importante a la  hora de tratar con datos muchos criminales intentan robar datos para hacer de ellos un beneficio es por eso que  un sistema tiene que tener medidas frente a ello.

Es por eso que usted podrá utilizar las siguientes tácticas para que su sistema  sea más seguro y tenga buena confiabilidad para sus usuarios.

Fecha Actualizada

31 de Abril de 2021

DETECTAR ATAQUES

  • Verificar integridad de mensajes/Valores Hash  para redundancia de archivos.
  • Detección de denegación del servicio/Comparación de tráficos  de red en un historial.

    • RESISTIR LOS ATAQUES

  • Identificar actores/Con IP, códigos de acceso y direcciones  para cada actor.
  • Autorizar actores/Que tenga derechos para  modificar o acceder a fuentes de datos.
  • Limitar accesos/En los que unos servicios si tengan acceso y otros no.

    • REACCIONAR A LOS ATAQUES

  • Revocar acceso/Cuando hay un ataque se reducirá el nivel  de entrada a diferentes recursos.
  • Bloquear computadora/Limitaciones de muchos intentos fallidos al acceder a un sistema.

    • RECUPERARSE DE ATAQUES

  • Restaurar/Mantener en reserva servidores de tal propósito.
  • Auditorías/Registro de acciones de usuarios y sistemas.

    • Escenario

    Muchas veces pueden ocurrir ataques debido a que las restricciones están muy  débiles como en el caso de que cualquier usuario pueda tener permisos para modificar datos y eliminar,  este tráfico sin límites puede llevar a un fallo de seguridad, una buena seguridad tendría limitaciones  de acceso  a un módulo o sistema de cualquier usuario.

    Ahora que ya tenemos una idea de un posible escenario a tratar clasifiquemos las partes más importantes de nuestro escenario:

  • Estimulo:Ataque de modificación de datos o robo por mal manejo de restricciones.
  • Respuesta:Limitar a usuarios, autorizar sus accesos de modificación de datos, manteniendo un control por tipos de roles o módulos.
