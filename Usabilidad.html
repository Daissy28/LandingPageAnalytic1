Usabilidad

Aprendibilidad, Eficacia, Recordabilidad y Satisfacción son algunos que pronostican una buena experiencia de usuario. Una buena arquitectura ayudará como su distribución de elementos y  colores o informaciones relevantes para el usuario dentro de una interfaz.

Estas son algunas de la tácticas más utilizadas de Usabilidad:

Fecha Actualizada

INICIATIVA DE USUARIO

  • Cancelar/El usuario presiona el botón de cancelar para que este  comando haga un liberado  dentro del sistema y se tomen medidas.
  • Pausar/Reanudar para operaciones de larga duración  el usuario tiene el privilegio de elegir  dependiendo de sus necesidades.
  • Agregado/Proporciona capacidad de agregar  objetos par no hacer monótono una operación.

    • INICIATIVA DE SISTEMA

  • Modelo de tarea/Determinar cuando el usuario necesitará asistencia.
  • Modelo de usuario/Comportamiento y conocimiento del usuario en el sistema.
  • Modelo de sistema/Modelo de sí mismo para saber su comportamiento y advertir al usuario.

    • Escenario

    Un usuario requiere de ayuda al utilizar una aplicación debido a que es su  primera vez en ella, por lo tanto no conoce sus métodos para configurar a su propio estilo,  se necesitará una asistencia del propio sistema determinando contextos que el usuario establezca  como gustos.

    Ahora que ya tenemos una idea de un posible escenario a tratar clasifiquemos las partes más importantes de nuestro escenario:

  • Estimulo:El usuario necesita de ayuda para explorar un poco más en el sistema.
  • Respuesta:Se proporciona un modelo de sistema para que este pueda hacer más fácil la experiencia en el sistema.
